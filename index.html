Endurance Data

  • GitHub Repo (client-side)
  • GitHub Repo (server-side)
  • Live App

Description: This app serves as a manual-entry workout log for endurance athletes (runners and cyclists). It utilizes user authentication, and user-data is persisted in a database. As a former collegiate athlete, I used to log my workouts in a notebook like a barbarian, so I built this app to computerize the process. Data-analysis feature using D3.js is on my to-do list. username: user password: password (If you try the 'Change Password' or 'Change Username' feature, please change them back!)

Tech-Stack: This is a full-stack React App. The state of the front-end is managed using a combination of Redux and stateful components, and the browser's local storage is utilized for JSON Web Tokens. Some other NPM packages that are used are redux-thunk, react-router, react-spinkit and Moment.js. The HTML is styled using CSS3 with a responsive and mobile-first design, and each component is tested using Enzyme and Jest. The app is continuously deployed on Netlify.

The back-end is written in Node.js and Express. Users' passwords are hashed using bcrypt before being stored in the database. All user data is persisted using MongoDB, hosted on mLab. Database queries are written using Mongoose, and I wrote my own middleware function to verify incoming JSON Web Tokens. The back-end utilizes Mocha & Chai for testing, TravisCI for continuous integration and Heroku for continuous deployment.
